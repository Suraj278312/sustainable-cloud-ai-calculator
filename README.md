# 🌱 Sustainable Cloud & AI Cost Calculator

## 📌 Overview

The **Sustainable Cloud & AI Cost Calculator** is a modern web-based application designed to analyze and optimize both **cloud computing costs** and **AI energy consumption**. As cloud platforms and artificial intelligence systems continue to grow rapidly, their impact on both operational expenses and environmental sustainability has become increasingly important.

This project aims to provide a simple yet powerful tool that helps users understand how cloud resources are billed and how AI systems consume energy based on token usage. By combining cost estimation with sustainability awareness, this application bridges the gap between financial efficiency and environmental responsibility.

The system is built using **HTML, CSS, and JavaScript**, making it lightweight, fast, and accessible without requiring any backend infrastructure. It provides real-time calculations, interactive charts, and actionable insights that can help developers, students, and organizations make smarter decisions.

## ☁️ Cloud Cost Calculation

Cloud computing services are typically billed based on three major components:

### 1. Storage Cost

Storage refers to the amount of data stored in the cloud (e.g., images, files, databases).

**Formula:**

```
Cost = Storage (GB) × Price per GB
```

### 2. Compute Cost

Compute cost is based on how long virtual machines or servers are running.

**Formula:**

```
Cost = Hours × Cost per Hour
```

### 3. Data Transfer Cost

This includes the cost of transferring data from the cloud to users (outbound traffic).

**Formula:**

```
Cost = Data Transfer (GB) × Rate
```

---

### 💡 Example Calculation

If a user enters:

* Storage: 100 GB
* Compute: 10 hours
* Data Transfer: 50 GB

The system calculates total cost based on selected provider pricing.

## 🤖 AI Token Energy Calculator

Artificial Intelligence systems process data in the form of **tokens**. Each token represents a small unit of text, and processing more tokens requires more computational power.

### ⚡ Key Concept

* More tokens = More computation = More energy consumption
* Larger prompts = Higher carbon footprint

---

### 🧮 Token Estimation Logic

The application estimates tokens using:

```
Tokens ≈ Words × 1.3
```

---

### 📊 Energy Levels

| Token Count | Energy Usage |
| ----------- | ------------ |
| 0–50        | Low          |
| 50–150      | Medium       |
| 150+        | High         |

---

### 💡 Example

**Bad Prompt:**
"Explain everything about artificial intelligence in detail"

**Good Prompt:**
"Explain AI basics in 5 points"

👉 Shorter prompts reduce energy usage and improve efficiency.
